## コードの解説

このセクションでは、サンプルコードを上から順番に貼るのではなく、「何が大事か」を中心に説明します。  
初学者がつまずきやすい部分は、コードの意味とあわせて補足すると分かりやすくなります。  
コードや本文で初めて登場する主要な API、メソッド、設定値は、「何をするものか」「なぜここで使うのか」「実行すると何を確認できるのか」を短く説明してください。

例:

### 1. 最初の設定

ここでは、`[対象技術名]` を使うための初期設定をしています。  
この設定がないと、`[機能名]` を正しく利用できません。

### 2. `[機能名]` の実装

この部分では、`[入力]` を受け取り、`[出力]` を返しています。  
最初は細かい最適化よりも、「どこで何をしているか」が追えることを優先して読むのがおすすめです。

API 説明の例:

- `click()`: 対象のボタンやリンクをクリックする操作です。この例では、ユーザーが画面上で押す操作をテストから再現します。
- `fill()`: 入力欄に文字列を入れる操作です。この例では、フォームにテスト用の値を入力します。
- `toBeVisible()`: 対象の要素が画面に表示されていることを確認します。この例では、操作後に期待した表示が出ているかを検証します。

### 3. 実行結果の確認

このコードを実行すると、`[期待する動作]` を確認できます。  
もし動きが違う場合は、設定値やファイルの配置を先に見直してください。

## よくあるエラーと対処方法

このセクションでは、初学者が止まりやすいポイントを先回りして書きます。  
エラー文そのものよりも、「なぜ起きやすいか」と「どう直すか」が分かる形にするのが大切です。

例:

- `Module not found` と表示される  
  依存関係が入っていない可能性があります。まずはルート `README.md` の共通セットアップを確認し、そのうえでこのチュートリアル固有の追加インストールが抜けていないか見直してください。

- ポート番号が競合して起動できない  
  すでに別のアプリが同じポートを使っている可能性があります。ポート設定を変えるか、不要なプロセスを停止してください。

- コマンドが見つからない  
  実行環境が未インストールか、パスが通っていない可能性があります。ルート `README.md` の共通セットアップ手順と、対象技術の公式ドキュメントを確認してください。
