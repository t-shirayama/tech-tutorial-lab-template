# Getting Started の学習メモ

## 要点

- 最初にルート `README.md` を読み、全体の構成を確認する
- 共通セットアップは一度だけ行い、各チュートリアルでは追加作業だけ確認する
- `docs/roadmap.md` を見ると、学習順序が分かりやすい

## 補足

- いきなり細かい実装に入る前に、どのファイルが何の役割を持つかを見ると進めやすい
- `templates/` は新しいチュートリアルを書くときの見本として使う

## つまずきやすい点

- ルート `README.md` を読まずに各チュートリアルから読み始めると、全体像が見えにくい
- 共通セットアップと各章の追加準備が混ざると、同じ作業を何度も書いてしまいやすい

## 復習ポイント

- 共通セットアップはどこにまとめるのか
- `tutorials/`、`docs/`、`examples/`、`templates/` はそれぞれ何のためにあるか

## あとで見返したいこと

- 対象技術向けに最初に追加すべきチュートリアル案
- 共通セットアップに含めるべきツールや依存関係
